# Product

## Register

product

## Users

People who want to play 10x10 Gomoku against a browser-hosted AlphaZero-style model and inspect how the model thinks. They are usually in an exploratory play, learning, or demo context, not a marketing browse.

## Product Purpose

Provide a deployable GitHub Pages game interface that lets a user play, request a hint, inspect candidate moves, compare neural-network intuition with MCTS search, and understand the principal variation without needing a backend server.

## Brand Personality

Focused, analytical, calm.

## Anti-references

Avoid toy-game chrome, decorative dashboard clutter, unexplained AI jargon, dense technical panels that obscure the board, and visual effects that make the search harder to read.

## Design Principles

- Explain the model by showing its decisions, not by adding long prose.
- Keep the board primary; analysis should support play rather than compete with it.
- Prefer clear state and consistent controls over novelty.
- Make search depth feel visible without drawing the entire tree.
- Treat the static browser build as a real product surface, not a demo scaffold.

## Accessibility & Inclusion

Target WCAG AA contrast for interface text, visible keyboard focus, reduced-motion alternatives for animated search effects, and resilient layouts for Chinese labels and longer translated UI copy.
